# 研究协议

## 主链路

研究必须围绕一项用户需要做出的业务决策展开：

`决策 -> 研究问题 -> 可证伪假设 -> 方法组合 -> 证据 -> 结论 -> 行动建议`

先确认决策对象、期限、市场、时间口径、范围、排除范围和资源限制。用户已经明确的信息不重复询问；缺失信息会改变结论或研究成本时写为「待确认」并向用户提问。

## 研究模式

- 机会扫描：识别现在发生了什么变化，判断继续、停止或补证。
- 标准研究：默认模式，按需覆盖市场、用户、竞品、商业可行性和公司匹配。
- 深度尽调：加入付费数据、专家或用户访谈、单位经济模型和关键假设实验；额外成本和权限必须先批准。

## 假设与方法

把方向拆成会改变决策的假设。每项假设至少记录：支持理由、所需证据、证伪条件、当前状态和下一步验证。

方法必须匹配问题。市场规模使用多口径测算；用户原因使用行为、访谈或评论证据；竞品增长结合时间序列、产品变化和获客信号；商业模式结合价格、支付链路和单位经济；进入能力结合现有资产、缺口与实验。

## 结论与行动

结论必须区分事实、推断、假设和建议。推荐必须同时说明做什么、不做什么、为什么、首个验证动作、指标、验证周期、放大条件和停止条件。

研究完成不等于方向自动成立。只有用户明确批准后，推荐才能成为项目方向并进入产品定义阶段。
