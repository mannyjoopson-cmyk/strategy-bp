# Contributing

Contributions should preserve the plugin's decision-first, evidence-traceable, and user-approved boundaries.

Before submitting a change:

1. Keep the core Skill concise; put detailed methods in focused reference files.
2. Do not add product assumptions, fabricated evidence, automatic approval, outreach, purchasing, or personal-data collection.
3. Update or add an example when changing research routing or evidence behavior.
4. Run `python3 scripts/validate.py` from the repository root.
5. Explain the problem, behavior change, risks, and verification evidence in the pull request.
